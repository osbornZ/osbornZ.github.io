混编配置

Swift访问OC: 只需要在桥接文件 ProductName-Bridging-Header.h 中导入需要暴露给Swift的OC类，即可在Swift中访问相应OC类及方法。

创建swift 文件时候，xcode 会提示创建 桥接文件。 xxx-Bridging-Header.h

OC访问Swift: 在OC类中导入 ProductName-Swift.h 文件（Target名称不同，对应文件名称也不同，工程配置中能看到），即可访问Swift中暴露给OC的属性和方法。

build 会系统生成供oc调用的 .h，包含类及 public 接口属性。

Swift 的 nil 和 Objective-C 中的 nil 并不一样。在 Objective-C 中，nil 是一个指向不存在对象的指针。在 Swift 中，nil 不是指针——它是一个确定的值，用来表示值缺失。任何类型的可选状态都可以被设置为 nil，不只是对象类型。

项目复盘

发表于 2020-04-15 | 分类于 项目管理 |

一个阶段性的工作，必须需要一个复盘性的回顾会议。总结过往的经验教训，寻求好的解决方法，为团队组织建设提供保障。

PDCA循环 （Plan、Do、Check、Act）

那么，今年由于疫情一直在家办公这个方式 松懈了自己 影响了团队, 我们更应该及时同步信息、回顾问题、优化效率。

什么时候应该复盘

问题的出现往往不会等人，而且有时候会接二连三的频发，这时候我们需要立即回顾，当然这样的场景一般不会存在。

  • 每天的站会同步、
  • 迭代周期一般在一到两周、

所以，复盘可以在任何时候。

具体复盘什么内容

  • 关键事件、某个问题
  • 里程碑节点

团队方面，可以优化管理机制和项目流程，可以对某一类问题达成共识，以后遇到时应该怎么解决；
个人方面的总结比较灵活，方向可以参考述职报告。

最好是问题干系人，事件影响者，项目组成员尽量都参加；视问题程度可上升一到两级领导。

复盘步骤

  • 确定会议主题和范围
  • 与会人员和会议主持人
  • 准备好材料，或相关同学发言阐述
  • 讨论和挖掘问题本质
  • 总结周知
  • 跟进实施

注意

  1. 针对问题，避免对人
  2. 多进行自我批评

个人回顾，个人梳理，个人总结，谨记谨记。
